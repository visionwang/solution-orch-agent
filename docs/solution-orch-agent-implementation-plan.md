# 需求到投标智能方案生成 Agent 实施计划

## Summary

- 一期目标是端到端 Web 工作台原型：上传需求、招标和产品资料，自动抽取需求，匹配产品能力，生成可编辑方案/标书草稿，并输出 AI 审核意见。
- 技术栈采用 TypeScript、Mastra Workflow、Vue3/Vite SPA、Node 原生 SQLite 和本地文件存储。
- 交付策略是先跑通可演示闭环，再逐步替换更强的文档解析、向量检索和模型调用能力。

## Implementation Steps

1. 初始化工程：创建 Vue3/Vite 前端、TypeScript 后端、Mastra 目录、环境变量示例和启动脚本。
2. 建立数据与文件层：用 SQLite 管理项目、文档、需求项、知识片段、匹配结果、草稿、审核意见和工作流记录。
3. 实现核心 Workflow：文档解析、需求抽取、产品知识入库、产品匹配、方案/标书生成、AI 审核。
4. 实现 API：创建项目、上传文档、运行流程、读取结果、保存草稿、导出 DOCX。
5. 实现 Vue3 工作台：项目列表、详情、上传、需求清单、匹配结果、草稿编辑、审核意见和导出入口。
6. 补充测试与验收：覆盖解析、抽取、匹配、生成、审核和前端主流程。

## Test Plan

- 单元测试：文档解析、需求结构化、匹配评分、敏感信息脱敏、导出内容生成。
- 集成测试：创建项目、上传样例文档、运行 Workflow、查看需求/匹配/草稿/审核结果。
- 前端验证：Vue3 页面能完成新建项目、上传文档、运行流程、编辑草稿和导出。
- 安全检查：日志、页面和导出内容不包含 API Key、环境变量和服务端本地路径。

## Assumptions

- OpenAI 兼容接口通过环境变量配置，原型默认可在没有模型密钥时使用确定性规则生成可演示结果。
- Word 支持以 `.docx` 为主；`.doc` 和扫描件 OCR 不纳入一期。
- SQLite 使用 Node 24 的 `node:sqlite`，避免额外原生依赖。
- DOCX 导出以基础结构和可编辑内容为主，不实现复杂模板排版。
